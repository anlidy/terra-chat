# Product

> **Status**: Active
>
> **Scope**: FurryChatbot product positioning and experience principles
>
> **Last verified**: 2026-07-22

## Register

product

## Platform

web

## Users

Individuals doing ongoing research, learning, product, client, or operational work. They return to related source files and conversations over time and need context to remain organized without repeatedly uploading the same material.

## Product Purpose

FurryChatbot combines configurable AI conversations, multimodal work, and document-backed retrieval. Success means users can create a durable project, add trusted source files once, and use those files across multiple focused conversations while independent chats remain fast and lightweight.

## Positioning

One adaptable AI workspace where conversations, generated work, and reusable source knowledge stay connected.

## Brand Personality

Calm, capable, and direct. The interface should feel focused during long desktop sessions and remain clear when users are checking uploads or navigating on mobile.

## Anti-references

Avoid decorative AI-dashboard styling, nested card grids, excessive rounding, gratuitous gradients, hidden file states, and project navigation that obscures independent chats.

## Design Principles

- Keep the current task visually primary.
- Make knowledge scope and ingestion state legible.
- Use familiar product navigation and standard controls.
- Preserve user work during organization changes.
- Prefer restrained hierarchy over decoration.

## Accessibility & Inclusion

Target WCAG 2.1 AA contrast and keyboard operation. Touch targets should reach 44×44 px on coarse pointers, motion must respect reduced-motion preferences, and status cannot rely on color alone.
